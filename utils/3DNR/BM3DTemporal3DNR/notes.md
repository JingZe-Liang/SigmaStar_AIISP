# BM3D-temporal 3DNR 算法说明

## 1. 算法定位

这份代码里使用的 3DNR 并不是单个“纯 VBM4D / 纯 VNLB”算法，而是一个**两阶段混合式传统 3DNR**：

**时域对齐 + 时域融合 + BM3D 空间去噪**

也就是说，它先利用相邻帧之间的时间冗余降低噪声，再把融合后的结果交给 BM3D 做进一步的空间域精修。

主流程可以概括为：

1. 读取目标帧及其前后若干帧
2. 将 raw 图像归一化到 `[0,1]`
3. 自动估计噪声强度 `sigma01`
4. 用光流把相邻帧对齐到目标帧坐标系
5. 对齐后做自适应时域融合
6. 对融合结果做 BM3D 去噪
7. 回到 raw 数值域保存，并计算 raw 域指标

从定义上看，它之所以属于 3DNR，是因为它同时利用了：

- 空间维度 `x, y`
- 时间维度 `t`

所以它处理的不是单帧图，而是一个小时域视频片段。

---

## 2. 整体思路

这套方法的核心思想是：

> **先用时间信息降低随机噪声，再用单帧强空间去噪方法做最后精修。**

可以把它理解成一个工程化、易复现的传统视频去噪方案。

相比只对一张 noisy 图做 BM3D，它多利用了相邻帧；相比完整 VBM4D / VNLB，它又更容易直接用 Python 跑起来。

---

## 3. 第一步：读取时域序列

代码会根据目标文件名，例如：

- `frame5_noisy3.tiff`

去同一个文件夹里找同一 `noisy_id` 的前后相邻帧，比如：

- `frame2_noisy3.tiff`
- `frame3_noisy3.tiff`
- `frame4_noisy3.tiff`
- `frame5_noisy3.tiff`
- `frame6_noisy3.tiff`
- `frame7_noisy3.tiff`
- `frame8_noisy3.tiff`

默认参数是：

- `radius = 3`

这表示会尽量收集目标帧前后各 3 帧，总共最多 **7 帧**。

这样做的目的，是让算法有足够的时间冗余用于降噪。

---

## 4. 第二步：raw 归一化

原始 TIFF raw 图像通常不是 `[0,1]`，而是例如：

- 10-bit：`0 ~ 1023`
- 12-bit：`0 ~ 4095`

所以代码会先按下面的公式归一化：

\[
x = \frac{img - black\_level}{white\_level - black\_level}
\]

然后裁剪到 `[0,1]`。

这样做的好处是：

1. 后续时域融合与 BM3D 都能在统一数值范围上工作
2. PSNR 计算时可以把 `data_range` 固定成 `1.0`
3. 参数 `sigma01` 更直观，表示归一化域下的噪声标准差

最后在保存结果时，再反变换回 raw 数值域。

---

## 5. 第三步：自动估计噪声强度 `sigma01`

如果你没有手动指定 `sigma01`，代码会自动估计。

估计方法是：

1. 把目标帧转成灰度图
2. 做一个小高斯模糊
3. 用原图减去模糊图，得到高频残差
4. 用 **MAD（Median Absolute Deviation）** 来估计噪声标准差

最后把估计值裁剪到一个合理范围，例如 `[1e-4, 0.25]`。

这个 `sigma01` 是整套算法最关键的参数之一，因为它同时影响：

- 时域融合时的像素相似性权重
- BM3D 的去噪强度

如果 `sigma01` 设得太小：

- 去噪不够强
- 时域融合更保守

如果 `sigma01` 设得太大：

- 去噪更强
- 但更容易糊、丢失细节

---

## 6. 第四步：光流对齐

这是 3DNR 最关键的一步之一。

如果直接把多帧拿来平均，而不做运动对齐，那么：

- 静止区域能降噪
- 运动区域一定会出现明显拖影

所以代码先使用 **Farneback 稠密光流**，把每一帧都对齐到目标帧坐标系。

具体做法：

1. 目标帧转灰度，作为参考
2. 当前帧转灰度，与目标帧做光流估计
3. 得到从目标帧到当前帧的位移场 `flow`
4. 用 `cv2.remap()` 把当前帧 warp 回目标帧坐标系

这样，对齐后的多帧中：

- 静止物体会尽量落到同一位置
- 时域融合时就可以更安全地利用它们

### 为什么需要这一步

因为 3DNR 的本质就是利用“同一个场景点在不同时间上反复被观察”的冗余信息。

只有把这些对应位置对准，时间维上的平均和统计才有意义。

---

## 7. 第五步：时域融合（真正的 3DNR 主体）

对齐完成后，代码并不是简单做均值，而是做了一个**自适应时域融合**。

### 7.1 加权均值

对于每个像素，先计算它在每一帧和目标帧之间的差值：

\[
diff = I_t - I_{target}
\]

再根据差值大小构造一个高斯型权重：

\[
w = \exp\left(-\frac{diff^2}{2(temporal\_sigma\_scale \cdot sigma01)^2}\right)
\]

含义是：

- 和目标帧越接近，权重越大
- 差得越多，权重越小

因此：

- 静止区域更容易被多帧共同利用
- 运动区域、不一致区域自动被压低权重

代码还会把目标帧自己的权重乘以 `2.0`，目的是：

- 防止过度依赖别的帧
- 防止结果过度平滑
- 减少拖影

### 7.2 中值融合

除了加权均值，代码还计算了一个 **temporal median（时域中值）**。

时域中值的作用是提高稳健性：

- 对异常帧更不敏感
- 对局部对齐失败更稳
- 对局部闪烁或误匹配更鲁棒

### 7.3 最终融合

最后结果不是只取均值，也不是只取中值，而是二者混合：

\[
fused = mean\_weight \cdot fused\_mean + (1 - mean\_weight) \cdot fused\_med
\]

默认：

- `mean_weight = 0.7`

这意味着：

- 主要依赖均值提高去噪能力
- 再引入一部分中值，提高稳健性

### 这一层的本质

这一步就是这份代码里真正承担 **“时域降噪”** 的主体。

可以把它理解为：

> **一个偏向静止区域提分、同时尽量抑制运动错误传播的多帧融合器。**

---

## 8. 第六步：BM3D 空间去噪

经过时域融合后，随机噪声已经被压低了一大截，但图像里通常还会残留：

- 细小的空间噪声
- 局部纹理附近的噪点
- 时域融合留下的细碎残差

于是代码再调用 **BM3D**：

\[
\text{denoised} = \text{BM3D}(fused, sigma01)
\]

BM3D 是传统图像去噪里非常经典的强基线，它的核心思想是：

1. 找到相似 patch
2. 把它们堆成 3D 组
3. 在变换域做协同滤波
4. 再重建回图像

它特别适合：

- 单帧空间噪声去除
- 保留一定结构和纹理

所以这一步的角色是：

> **把时域融合后的中间结果，再做一遍强力的空间精修。**

---

## 9. 为什么这套方法算 3DNR，但又不是严格的 VBM4D / VNLB

这点需要说清楚。

### 它确实属于 3DNR

因为它利用了：

- 时间维 `t`
- 空间维 `x, y`

而不是只看一张图。

### 但它不是完整论文版 VBM4D / VNLB

因为：

- VBM4D 通常是更完整的时空块匹配 + 4D 协同滤波
- VNLB 是基于时空 patch 统计和贝叶斯估计的更完整框架

这份代码没有把整个时空块组一次性送进完整论文级的联合协同滤波器，而是采取了一个更实用的工程方案：

- 时间维：光流 + 融合
- 空间维：BM3D

所以这份代码更准确的叫法应该是：

**BM3D-temporal 3DNR pipeline**

它的优点是：

- 更容易直接 Python 复现
- 参数更少
- 调试更容易
- 不依赖 MATLAB / CUDA / 特殊编译环境

---

## 10. 轻微拖影是怎么来的

在这份代码里，拖影并不是专门强加出来的，而是 3DNR 的天然副作用被控制在一个较弱水平。

因为：

- 时域融合会尽量利用相邻帧
- 如果运动区域没有完全对齐
- 那么它就会带来一点点拖影

但代码通过以下手段在抑制这种拖影：

1. 光流对齐
2. 差异大的像素权重自动减小
3. 目标帧权重额外提高
4. 时域中值增强稳健性

所以它不是“故意制造强拖影”，而是：

> **在尽量保住真实 PSNR 的前提下，允许时域融合保留少量自然的时间痕迹。**

---

## 11. raw 域指标是怎么测的

这份代码里，最终的数值评估仍然是在 **raw 域** 完成的，而不是在 RGB 显示图上测。

### 评价流程

1. noisy / denoised / clean 都按同样方式归一化到 `[0,1]`
2. 构造一个布尔 mask
3. mask 中间区域设为 `False`
4. 只在外圈区域计算：
   - `raw_MSE`
   - `raw_PSNR`
   - `raw_SSIM`

### 为什么只测外圈

这是你之前要求的设定：

- 去掉中间大块区域
- 只评估外圈

这可以避免把你不关心的中心区域混进统计里。

### 为什么说这些数值是真实的

因为当前这版代码里：

- **没有保底器**
- **没有使用 clean 反向修正 denoised**
- **没有人为把 denoised 和 noisy 混起来刷分**

所以：

> 现在输出的 PSNR / SSIM 就是算法原始结果的真实数值。

---

## 12. RGB 显示和 raw 指标的关系

你后面又把显示改成了 RGB 效果图，但指标前缀改成 `raw_`。

这两者之间的关系是：

- **raw 指标**：真实评价数值，严格在 raw 域算
- **RGB 图像**：只是为了看起来更直观

也就是说：

- 去噪是在 raw 域完成的
- MSE / PSNR / SSIM 也是 raw 域算的
- 最后那张 RGB 图只是做显示

这样做的好处是：

- 数值不被 demosaic / gamma / 色彩映射污染
- 视觉展示又更符合人眼习惯

---

## 13. 关键参数说明

### `radius`
控制时间窗口大小。

- `radius=3` 表示最多使用 7 帧
- 更大：时域冗余更强，但对齐难度也更大
- 更小：更稳，但利用的时间信息少

### `sigma01`
控制整体噪声强度。

- 越大：去噪更强，也更容易糊
- 越小：保守，细节更稳，但提分可能不足

### `temporal_sigma_scale`
控制时域融合时对差异的容忍度。

- 大：愿意融合更多帧，可能提分更高，也更可能拖影
- 小：更保守，更依赖目标帧本身

### `mean_weight`
控制加权均值和时域中值的比例。

- 大：更偏均值，降噪更强
- 小：更偏中值，更稳健

### `black_level / white_level`
决定 raw 归一化范围。

这个如果设错，会直接影响：

- raw_MSE
- raw_PSNR
- raw_SSIM

所以必须跟你的 raw 位深一致。

---

## 14. 这套算法的优点

1. **不是深度学习**，不需要训练数据
2. **可直接 Python 运行**
3. 结构清楚、容易解释
4. 兼顾：
   - 时间域降噪
   - 空间域精修
5. 结果是真实 raw 指标，不造假

---

## 15. 这套算法的局限

1. 对齐依赖光流质量
2. 高速运动、遮挡区域可能融合不好
3. 它不是完整论文版 VBM4D / VNLB
4. 时域融合效果很依赖参数是否合适

所以这套方案更适合：

- 低到中等运动
- 静止区域较多
- 需要直接落地、稳定复现的传统 3DNR 场景

---

## 16. 一句话总结

**这份代码里的 3DNR，本质上是“先用光流把相邻帧对齐到目标帧，再做自适应时域融合，最后用 BM3D 对融合结果做空间精修”的非深度学习传统视频去噪方案。**

### 17.5 最常用参数怎么调

#### 想更稳一点

```python

```

特点：

- 更保守

#### 想更强一点

```python

```

特点：

- 融合更激进
- 去噪更强

### 17.6 `white_level` 要特别注意

如果你的 raw 不是 12-bit，就不要默认用 `4095`。

常见设置：

- 10-bit：`white_level=1023`
- 12-bit：`white_level=4095`

这个参数会直接影响 raw 域的 PSNR / SSIM 数值。

---

### 17.7 如果颜色显示不对

如果你后面那版 RGB 显示脚本里颜色不对，通常不是去噪错了，而是 **Bayer pattern** 设错了。

你可以尝试在显示部分切换 Bayer pattern，例如：

- `RGGB`
- `BGGR`
- `GRBG`
- `GBRG`

指标依然还是 raw 域，不受这个显示参数影响。

---

### 17.8 如何判断当前结果是不是“真数据”

当前这版代码里：

- 不会用 GT 回拉结果
- 不会把 denoised 和 noisy 混合刷分
- 不会人为保证 PSNR 一定超过 noisy

所以只要你看到：

- `raw_Denoised PSNR < raw_Noisy PSNR`

那就表示：

> 当前这组参数下，算法真实没有提上去。

这正是“真数据”的含义。
