## 伪影和空洞

这两种现象都是 **“运动检测（Motion Detection）失败”** 导致的，但在视觉表现和成因上完全不同。

#### 1. 拖影 / 伪影 (Ghosting / Motion Blur) 的成因

- **物理成因**：在极低照度下，Sensor 产生的散粒噪声（Shot Noise）极大。如果系统把“运动阈值”设得太低，满屏的噪点都会被当成“运动物体”，导致降噪失效。为了强行降噪，ISP 只能**拉高运动阈值**或**强制施加时域权重**。
- **结果**：当人走过时，系统为了降噪，依然保留了 30%~50% 的历史帧。人的每一次移动，都会把上一个瞬间的影子留在“历史累积帧”里，层层叠叠，最终拉出一条长长的、类似多重曝光的“彗星尾巴”。

#### 2. 空洞 / 撕裂 (Holes / Tearing) 的成因

- **物理成因（非常经典的工业难题）**：这被称为 **“平坦区域陷阱 (Flat Area Trap)”** 或 **“孔径问题 (Aperture Problem)”**。
- **具体表现**：假设一个穿着**纯黑色羽绒服**的贼在夜里走动。
  - **衣服边缘**：贼的轮廓盖住背景时，像素发生剧烈变化，系统成功检测到运动，轮廓瞬间更新（清晰无拖影）。
  - **衣服中间**：贼的肚子部分全是纯黑色，上一帧是黑的，这一帧还是黑的，像素差极小。系统**误以为肚子这块区域是“静止背景”**，于是施加了极大的时域降噪。
- **结果**：贼的边缘在往前走，但他的肚子和胸口却“黏”在了过去的时空里没有更新。在视觉上，就像是**人的躯干被撕裂了**，或者**肚子上破了一个大洞，透过洞看到了几帧之前的背景残影**。



## 星宸科技

 

星宸科技是脱胎于联发科（MStar）的全球安防监控芯片（IPC SoC）龙头企业，其底层的 ISP 调试逻辑在安防和车载领域具有绝对的代表性。

以下是关于星宸科技传统 3DNR 的原理、缺陷成因，以及其最新演进方向的详细解析

### 一、 星宸科技传统 3DNR 的底层工作原理

在星宸科技传统的 SoC 芯片（如 SSC335、SSC338Q 等型号）中，其 3DNR（在官方文档中常简称为 **NR3D**）主要依赖**运动与亮度自适应的 Alpha 混合算法 (Motion & Luma Adaptive Alpha Blending)**。

需要

根据星宸科技开发者社区（Comake Online）公布的《ISP API Tuning SOP》（图像调试标准作业程序）官方文档，其 3DNR 的核心受以下几个关键参数控制：

1.  **亮度与运动联合掩码 (`AREn` 参数)**
    *   **官方定义**：`AREn` 是一个开关参数，系统会“根据亮度（Luma）与运动（Motion）信息，限制 NR3D 强度”。
    *   **原理解析**：硬件 ISP 会计算前后两帧的像素差，并结合当前环境的亮度。如果像素差大，判定为“运动”，降低 3DNR 强度以防止拖影；如果差值小，判定为“静止背景”，加大 3DNR 强度以抹平噪点。
2.  **运动部分的补偿机制 (`BlendOffset` 参数)**
    *   **官方定义**：“决定写回 3DNR 参考帧，对于 motion 的部分，要补偿多少 3dnr 强度的控制。值越大，补偿的 3dnr 比例越多。”
    *   **原理解析**：由于单纯关闭 3DNR 会导致运动物体上充满噪点，星宸引入了 `BlendOffset`。这相当于强行在“运动物体”上混合一部分历史帧的降噪结果，但这也就是产生拖影的原因。
    *   **资料来源**：[SigmaStarDocs: ISP API Tuning SOP](https://wx.comake.online/doc/DD22dk2f3zx-SSD21X-SSD22X/customer/development/software/Px/zh/camera/ISP%20API%20tuning%20SOP.html)

### 二、 为什么会产生灾难性拖影和空洞？

1.  **极低照度下的巨大拖影成因**
    *   星宸 SOP 文档中提到了 `ARLumaTh`（亮度阈值）参数：“当环境亮度 Luma 极低时，NR3D 强度保持不变”。在极暗环境下（如星光级安防），由于 Sensor 散粒噪声极大，硬件运动检测会被噪声干扰失效。为了压制噪点，工程师只能被迫拉高全局的 3DNR 权重，这直接导致了移动物体出现严重的彗星尾状拖影。
2.  **空洞与穿透现象的成因**
    *   当传统的基于像素差值的 3DNR 处理大面积纯色运动物体（如穿黑衣服的人）时，由于衣服内部像素变化极小，星宸的硬件掩码会陷入“平坦区域陷阱”，误把移动的衣服当作静止背景处理，从而产生空间撕裂和空洞。在星宸官方的技术展示中，他们将这种传统 3DNR 的缺陷称为**“运动时候的拖尾和穿透现象”**。
    *   **资料来源**：星宸科技开发者平台发布的《视觉盛宴！星宸科技2023技术展示专场，全面展示智慧视觉领先成果》一文明确指出了传统画质的极限及穿透问题。



## 工业3dnr

在工业界芯片的真实图像信号处理（ISP）流水线中，所谓的“3DNR”其实就是**串联在 2DNR 之后，依靠运动检测（Motion Detection）来分配 2D 与历史帧权重的 动态融合器（Blender）**（图1来自ASICFPGA: Advanced 2D+3D Noise Reduction Core 架构说明，图2来自嘉楠科技调优指南）

![image-20260510194440866](https://images.cnblogs.com/cnblogs_com/blogs/760158/galleries/2506204/o_260515172747_image-20260510193801476.png)

- **noise_level（运动检测阈值）**：
  这是判定运动的标尺。芯片将当前帧与 DDR 里的历史帧相减，差值大于 noise_level，被判定为“运动区”；小于它，被判定为“静止区”。

- **blend_static（静止区融合权重）**：
  原话：“代表的是静止区域 2DNR 处理过的帧与 tnr 输出帧的权重”。
  **解读**：在背景等静止的地方，系统更信任时域（TNR），因此给历史帧的权重极高，2DNR 帧的权重极低，这样画面最干净。

- **blend_motion（运动区融合权重）**：
  原话：“代表 2DNR 处理过的帧与 tnr 输出帧的权重...会影响去噪强度”。
  **解读**：在人走动的地方，系统为了防止拖影，会大幅度提高“2DNR 处理过的当前帧”的权重。

- **blend_slope（过渡段斜率）**：
  在绝对静止和绝对运动之间，做了一个线性的坡度过渡，防止画面出现生硬的马赛克拼缝。

  ![image-20260510193801476](https://images.cnblogs.com/cnblogs_com/blogs/760158/galleries/2506204/o_260515172720_image-20260510194440866.png)



3DNR 的核心算法就是一个很简单的公式：
**最终输出 = (权重) × 2DNR当前帧 + (1 - 权重) × DDR里的历史帧**
这个“权重”，就是根据运动检测的结果，在 blend_static 和 blend_motion 之间来回切换

### 如何解释“拖影”和“空洞”？

#### 1. 为什么会有拖影？（blend_motion）

当人在运动时，按理说应该 100% 只用 2DNR 当前帧（抛弃历史），但低端 ISP 的 2DNR 效果太差，如果 100% 用它，人身上全是噪点。为了强行降噪，工程师只能把 blend_motion 调低，**强行在运动的人身上混入历史帧。这就是拖影的来源**

#### 2. 为什么会有空洞/人物变透明？（noise_level）

当一个人穿着纯色（如黑色）衣服移动时，衣服中间前后的像素差值极小，**导致它低于了 noise_level（运动检测阈值）**。
此时，芯片硬件**成了瞎子**，它把衣服中间误判成了“静止区”。
于是，人的边缘触发了 blend_motion（用了 2DNR，没有拖影），而人的肚子触发了 blend_static（死死锁在了历史背景上）。**中间没更新，边缘更新了，空间发生了撕裂，就形成了破洞**
